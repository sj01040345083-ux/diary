# 감정 일기 & 소비 습관 앱 구현 프롬프트 — "마음 한 줄 (A Line a Day)"

> 아래 내용을 Claude Code, Cursor, Lovable, Bolt, v0 등 AI 코딩 채팅창에 통째로 붙여넣어 사용하세요.

---

## Persona

당신은 세계 최고 수준의 UI/UX 디자이너이자 모바일 앱 기획자, React 개발자입니다. 사용자가 매일 3분 안에 감정과 소비를 기록할 수 있는 심플하고 감성적인 앱을 설계합니다. 복잡한 가계부가 아니라 **'마음을 기록하는 앱'**이 핵심이며, 소비 기록은 하루를 함께 돌아보는 보조 기능입니다.

## 앱 이름 (중요)

- 앱 이름: **마음 한 줄**
- 부제(태그라인): **A Line a Day**

앱 이름과 부제는 나중에 언제든 쉽게 바꿀 수 있어야 합니다. 이름을 코드 곳곳에 하드코딩하지 말고, `src/config/appConfig.ts` 파일에 `APP_NAME`, `APP_SUBTITLE` 상수로 분리해서 관리하세요. 화면·타이틀·알림 등 이름이 들어가는 모든 곳은 이 상수를 참조하도록 구현합니다.

## 수정 가능한 콘텐츠 분리 (중요)

나중에 코드 파일을 직접 열어 내용을 바꾸거나 추가할 수 있도록, 자주 바뀔 수 있는 데이터는 전부 `src/config/` 폴더 안에 별도 파일로 분리해서 관리하세요. 화면 로직과 데이터를 섞지 마세요.

- `src/config/appConfig.ts` — 앱 이름/부제 등 기본 설정
- `src/config/quotes.ts` — 명언 목록 배열. 객체만 추가/수정/삭제하면 앱에 바로 반영. 예: `{ id, text, author, category }`
- `src/config/categories.ts` — 소비 카테고리 목록 배열. 추가/삭제 시 입력 화면과 통계에 자동 반영
- `src/config/backgrounds.ts` — 배경 이미지 경로 목록. 이미지는 `src/assets/backgrounds/`에 넣고 경로만 지정하면 교체
- `src/config/emotions.ts` — 감정 이모티콘 목록

각 파일 상단에 "이 파일을 수정하면 어디에 반영되는지" 주석을 한 줄 달아 주세요.

## 테마/화면 설정 (사용자 변경 가능)

설정 화면에서 사용자가 직접 배경색·글씨체·글씨 크기를 고를 수 있게 구현하세요. 선택한 값은 각 사용자의 기기에 저장되어 다음에 앱을 열어도 유지됩니다.

- 배경색: 앱 컨셉에 맞는 3~5가지 중 선택 (아이보리, 연두, 베이지, 화이트, 연한 녹색)
- 글씨체: 가독성 좋은 3~4가지 중 선택 (Pretendard, SUIT, Noto Sans KR, Paperlogy)
- 글씨 크기: 보통 / 크게 / 아주 크게 (50대 이상 배려)

선택 가능한 목록은 코드(`config`)에서 관리하고, 그중 무엇을 쓸지는 사용자가 앱에서 고릅니다.

## Task

아래 조건을 만족하는 모바일 우선 앱을 **React + Vite + Tailwind CSS (PWA)**로 구현합니다. 한 번에 전부 만들지 말고 단계별로 파일을 생성하며 실행 가능한 코드까지 작성하세요.

1단계: Home 화면 → 2단계: 감정 기록 → 3단계: 소비 기록 → 4단계: 기록(달력) → 5단계: 리포트

## 핵심 컨셉

오늘의 감정을 기록하고, 오늘의 소비를 함께 남기며, 매일 조금씩 성장하는 나를 만나는 앱.

## 디자인 컨셉

- 분위기: 편안함·따뜻함·힐링·미니멀·감성
- 색상: 연두·녹색·아이보리·베이지·화이트
- 배경: 숲속을 산책하는 느낌의 수채화 스타일 연한 유채색, 자연광이 드는 느낌. UI를 방해하지 않게 투명도 15~20%
- 폰트: 두껍지 않고 따뜻하며 가독성 좋은 것 (Pretendard, SUIT, Paperlogy, Noto Sans KR)
- 버튼: 둥근 모서리, 큰 터치 영역, 은은한 그림자, 녹색 계열 Primary

## 첫 화면(Home) — 아래 순서대로 표시

1. **오늘의 명언** — 카드 형태, 하루 1회 자동 변경, 즐겨찾기(♡), 다른 명언 보기, 저장한 명언 모아보기. 명언 DB는 365개 이상 구성. ※ 명언에 실존 인물 출처를 붙일 경우 부정확할 수 있으니, 출처가 불확실하면 저자 표기 없이 넣거나 자체 창작 문구로 채웁니다.
2. **오늘 날짜** (예: 2026년 7월 2일 (목))
3. **오늘 기분은?** — 이모티콘 선택 😀😄😊😌😐😢😭😡😴😍😰, 선택 시 부드러운 애니메이션
4. **오늘의 한 줄** — 200자 제한, Placeholder "오늘은 어떤 하루였나요?"
5. **오늘 감사한 일(선택)** — 150자, Placeholder "오늘 감사했던 일을 적어보세요.", 별도 조회 가능
6. **오늘의 소비** — 카테고리(식비·교통·쇼핑·카페·문화·병원·생활·취미·기타), 금액·내용 입력, 여러 개 추가, 총합 자동 계산
7. **저장 버튼** — 큰 녹색 "오늘 저장하기"

## 하단 네비게이션 (5개)

🏠 홈 · 📖 기록 · ➕ 추가 · 📊 리포트 · ⚙ 설정

## 기록 화면

월별 달력에 감정을 이모티콘으로 표시. 날짜 클릭 시 그날의 감정·일기·감사·소비 모두 표시.

## 리포트 화면 (통계는 여기서만)

- 감정 통계: 이번 달 감정별 일수, 가장 많았던 감정
- 소비 통계: 도넛 차트, 카테고리별 비율
- 월 요약: 총수입·총지출·저축·하루 평균 소비·최다 소비 항목
- 다운로드: PDF·Excel·이미지
- ※ 홈에서는 통계를 크게 보여주지 않음

## 부가 기능

- 명언: 카테고리·즐겨찾기·검색
- 감사일기 모아보기
- 감정 분석: 월별·주간 추이, 긍정/부정/중립률
- 소비 분석: 요일별·주별·월별·평균
- 검색: 일기·감사·소비
- 백업: Google Drive·iCloud·로컬
- 알림: 매일 오후 9시 "오늘 하루는 어땠나요? 🌿", 시간 변경 가능

## 애니메이션

- 앱 실행 시: 숲에 바람 부는 부드러운 효과
- 저장 완료 시: 🌱 새싹이 자라는 효과
- 감정 선택 시: 이모티콘이 살짝 커졌다 돌아오는 효과

## 데이터 저장

로컬 우선(IndexedDB 또는 localStorage). 파일 상단에 타입 정의:

- `DiaryEntry`: date, emotion, oneLine, gratitude, expenses[]
- `Expense`: category, amount, memo
- `Quote`: id, text, author, category, isFavorite

## UI 원칙

- 여백을 충분히 사용
- 첫 화면은 감정 기록에 집중, 소비는 보조 배치
- 복잡한 통계는 리포트 화면에서만
- 50대 이상도 쉽게 쓰도록 크고 직관적인 버튼·입력창
- 앱을 열 때마다 부담이 아닌 편안함과 안정감을 느끼는 경험을 최우선으로
